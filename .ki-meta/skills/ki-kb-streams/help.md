# ki-kb-streams

Operate and govern the Streams zone of a Knowledge Islands base — the working copy of work in motion, run as the Enactment Process (the canonical change process: a proposal goes draft → ready → ratify → roll out → review → settle, and nothing reaches stable knowledge except through that gate).

**Invoke:** `ki-kb-streams audit | conform | help | educate | iterate | propose | ready | refresh | reject | review | rollout | settle`

**Modes:**

- `AUDIT`
- `CONFORM`
- `EDUCATE`
- `HELP`
- `ITERATE`
- `PROPOSE`
- `READY`
- `REFRESH`
- `REJECT`
- `REVIEW`
- `ROLLOUT`
- `SETTLE`

**See also:** For the five-zone model and note CRUD / routing use the `ki-kb` skill, which delegates the Streams zone here; for Markdown / TOML house style use `ki-authoring`.
