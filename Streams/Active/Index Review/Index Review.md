# Index Review

## Overview

A structured review of every index note under `Pillars/Knowledge Islands/`, working from the deepest leaf nodes up to `Knowledge Islands` itself. Each index note is reviewed against the index note standard - a prose Overview plus substantive H2 sections per direct child, each two to four sentences long - and enriched as needed before moving to the next level. The review is interactive: each note is announced before work begins, and decisions can be made through the Cowork UI before the changes are committed.

The stream also covers stream focus index tables (Active, Background, Dormant, Future, Settled) — verifying each `## Streams` table matches the top-level [[Streams/Streams|Streams]] table, and adding a recurring audit step to whichever maintenance activity owns structural health checks.

The goal is that by the end of the review, every index note in the Knowledge Islands pillar accurately contextualises its contents, and stream focus indexes are in sync and covered by ongoing maintenance.

---

## Governance

This stream follows the [[Pillars/Knowledge Islands/Governance/Processes/Enactment Process|Enactment Process]].

---

## Approach

Work leaf-to-root within each subtree. For each index note:

1. Announce the note being reviewed
2. Read the note and assess it against the index note standard in [[Pillars/Knowledge Islands/Governance/Conventions/Notes/Notes|Notes]]
3. Surface any decisions or options through the Cowork UI before making changes
4. Apply agreed changes
5. Move to the parent

Session boundaries are natural pause points - the review can be picked up at the next unreviewed note in the sequence.

---

## Sequence

The order follows the folder structure of `Pillars/Knowledge Islands/`, leaf nodes first within each subtree. To be confirmed at session start by reading the current folder structure.
