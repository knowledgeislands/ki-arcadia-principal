# Decision Records — Arcadia

Decision Records (DRs) capture significant standalone decisions made within the Knowledge Islands model as instantiated by Arcadia. DRs use the Nygard five-section format (Context, Decision, Consequences) with a `decision_type` taxonomy. Each type has its own prefix and its own serial sequence within the `KI-ARCADIA` scope.

Not every change warrants a DR. DRs record decisions with standalone standing: significant structural choices, adopted instruments, and cross-repo commitments. Trivial or implementation-only changes do not need one. A DR is a living present-state record: update it in place when the decision changes, with git retaining the history.

Records are ordered by reveal order — the logical sequence in which the decisions build on one another.
