# Reading Order

## Overview

The Knowledge Islands content can be entered from many directions. If you are reading for the first time, follow the sequence below — each step establishes the vocabulary and model that the next one builds on. Once familiar with the content, the wikilinks within each note are the navigational paths and any note can serve as an entry point from which to journey.

---

## Synopsis

### [[Concept]]

[[Concept]] establishes what Knowledge Islands is and situates it relative to Zettelkasten, PARA, and LLM-native wikis. Introduces the geographic metaphor and the role of Arcadia.  Expaning to cover the full model: what knowledge is (distinct from information and wisdom), the cycle of capture–connect–reflect, the island's geography (Capital, Library, Streams, Harbour), territories and archipelagos, agents, jurisdiction, and governance.

## Conventions

[[Conventions]] defines the shared language for Knowledge Islands: note format and structure, frontmatter properties and tags, folder layout, routing rules, and the enforced boundary between Pillars and Resources. Two sub-areas carry the most weight: Notes defines what a note is and how it is written - including the specialised types (collection cards, meeting notes, session digests, activity notes) that extend the base format. Structure defines where notes live - the top-level folder structure, routing rules, index note conventions, and the Pillars/Resources boundary.
