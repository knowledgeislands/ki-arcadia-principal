# Reading Order

The Knowledge Islands content can be entered from many directions. If you are reading for the first time, follow the sequence below — each step establishes the vocabulary and model that the next one builds on. Once familiar with the content, the wikilinks within each note are the navigational paths and any note can serve as an entry point.

- [[History of Knowledge Systems]] — why the problem of preserving knowledge is ancient, and what each era's solution reveals about the one that follows
- [[Concept]] — what Knowledge Islands is, its foundations in Zettelkasten, PARA, and LLM-native wikis, and the full model: the cycle, the island's geography, territories, agents, jurisdiction, and governance
- [[Notes]] — what a note is and why structure matters
  - [[Frontmatter]]
  - [[Format]]
  - [[Types]]
- [[Residency]] — where knowledge lives across the Library, canonical memory, and auxiliary memory
- [[Structure]] — the island's geography: Library, Harbour, and Streams
  - [[Library]]
- [[Enactment Process]] — how changes to the island are proposed and ratified
- [[Contribution Process]] — how knowledge crosses island boundaries
- [[Authoring Activities]] — how activities are designed, documented, and maintained across five layers
- [[Constitutional]] — the baseline activities required of any Knowledge Island
- [[Tending]] — the maintenance suite: health checks, rebuilds, reviews, and audits
- [[Briefings]] — daily automation that primes the island at the start of each working day
- [[Email]] — inbox management as a scheduled, repeatable process
- [[Linear]] — keeping the island aligned with the Linear project workspace
- [[Human]] — how the human agent operates within the island
- [[Agentic AI]] — the general patterns that apply across all AI agents operating within a Knowledge Island
- [[Agents/Claude/Claude|Claude]] — the primary AI agent: modes, memory architecture, and behavioural constraints
- [[Agents/ChatGPT/ChatGPT|ChatGPT]] — the secondary AI agent: read-heavy sessions and manual note routing
- [[Obsidian]] — the primary editor and vault environment; templates and vault configuration
- [[Tools/Claude/Claude|Claude]] — the Cowork connection, token economics, and activity prompt library
- [[Tools/ChatGPT/ChatGPT|ChatGPT]] — context-loading pattern and manual routing convention
- [[Tools/Linear/Linear|Linear]] — the MCP connection and scope boundary between Linear and the KI
- [[Microsoft 365]] — email, OneDrive, and Power Automate via the M365 MCP server

Once through the sequence, [[Knowledge Islands]] has a section for every area of the model — from there the wikilinks take you wherever you need to go.
